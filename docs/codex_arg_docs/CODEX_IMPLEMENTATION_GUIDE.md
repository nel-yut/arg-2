# Codex 実装ガイド

このプロジェクトは、検索導線で進行する ARG 用のフロントエンド実装です。Codex は以下の前提で作業してください。

## 最重要要件
- フレームワークは **Vite + React** を採用する
- ブログと本体サイトは **別ドメイン** を想定する
  - ブログ: `ino-blog.nel-app`
  - 本体サイト: 別ドメイン（名称未固定）
- ページ URL は **`/page-name-00000`** 形式
- `00000` はスプレッドシート A 列の **5桁数字文字列** を使う
- 検索は **完全一致のみ**
- 複数ワード検索は不可
- 人名は空白なし入力を前提とする
- 検索対象は **過去 Phase + 現在 Phase + 次 Phase の (1) のみ**
- LocalStorage に以下を保存する
  - `currentPhase`
  - `readPages`
- 既読状態は **ユーザーには見せない**
- すべてのページ右下に **小さく Phase とページ番号** を表示する
- ブログおよび `Phase4(31)` には検索UIを出さない
- `Phase4(31)` でも右下の Phase 表示は消さない

## ブログの扱い
- ブログは `Phase0` と `Phase-End` を持つ
- ブログは **本体と完全別レイアウト**
- ブログから本体への遷移は **本文中の直接リンクのみ**
- ブログに検索UIは表示しない

## スプレッドシートの扱い
ソースは `arg_route_v10_edit.xlsx` の `シート1 (1)` を使う。

現時点で実装で使う主要列は以下。
- A列: 作成するページ（将来的には 5桁ID を追加する運用）
- B列: 種別
- C列: ページの概要
- D列: コンテンツの内容（本文の元）
- E列: ページID（Phase 情報）
- F列: このページを検索するためのワード
- G列: コメント（画像プロンプト等）
- H列: コメント2（画像2枚目プロンプト等）

※ 実装時の変換スクリプトで列マッピングは明示的に定義すること。

## 実装方針
- まずは **静的 JSON 読み込み** で構わない
- スプレッドシート → JSON 変換スクリプトは後で実装する
- 先に UI / ルーティング / 検索 / Phase 解放ロジックを完成させる
- その後に import スクリプトを追加する

## 実装優先順
1. 型定義
2. ルーティング
3. LocalStorage 状態管理
4. 検索ロジック
5. 共通レイアウト
6. ブログ用レイアウト
7. ページレンダラ
8. JSON import 前提のデータ読み込み
9. スプレッドシート変換スクリプト

## やってはいけないこと
- 曖昧検索を入れない
- 部分一致を入れない
- 複数語検索を入れない
- 検索対象を「本文全文」から拾わない
- LocalStorage の phase 制御を飛ばさない
- 右下の Phase 表示を省略しない
