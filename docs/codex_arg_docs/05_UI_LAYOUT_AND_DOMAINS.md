# UI / レイアウト / ドメイン設計

## ドメイン
### ブログ
- ドメイン: `ino-blog.nel-app`
- 完全別レイアウト
- 本体サイトへの遷移は本文中の直接リンクのみ

### 本体サイト
- 別ドメイン（名称は実装時に設定）
- 一般HP + 資料室 + Ending を含む

## レイアウト方針
### ブログ
- 個人ブログ風
- コメント欄や記事一覧を持つ
- 404 風ページもブログの見た目で統一
- 検索UIなし
- 右下の Phase 表示はあり

### 一般HP
- 宗教団体の公開サイト風
- 白または淡色背景
- ヘッダあり
- 検索UIは右上
- 右下に小さな Phase 表示

### 資料室
- 古いHTML風
- 黒背景または暗色背景
- テキスト主体
- 黒塗り・欠損画像を許容
- 検索UIは右上
- 右下に小さな Phase 表示

### Ending / 特殊ページ
- 本体側の特殊スタイルでよい
- ただし右下の Phase 表示は維持

## 画像表示ルール
- 画像は 1 ページ最大 2 枚
- スプレッドシート G 列 = 画像1
- スプレッドシート H 列 = 画像2
- プロンプトはそのまま資産管理用に保持してよい
- 画像が不要なページは本文のみで表示する

## UI コンポーネント
### SearchBox
- 右上固定
- Enter で検索
- 検索結果ページに遷移

### PhaseBadge
- 右下固定
- かなり小さく控えめ
- 実装上はすべてのページで共通

### SearchResultsList
- タイトル一覧のみ
- タイトルクリックで詳細ページへ

## 404 / 未存在ページ
- 通常の 404 でもよい
- ブログドメインはブログらしい 404 を別で作ってもよい
