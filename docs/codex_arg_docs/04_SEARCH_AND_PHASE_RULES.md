# 検索とフェーズ解放ルール

## 検索ルール
- 完全一致のみ
- 1回の検索入力は1ワードのみ
- 複数ワード検索なし
- 部分一致なし
- 本文全文検索なし
- 比較対象は `searchWords` 配列のみ

## 人名検索のルール
- 人名は空白なし前提
- 結果が 0 件のときは次を表示する

```txt
検索結果が見つかりません。
複数ワード検索はできません。
人名は空白を入れず検索してください。
```

## 検索対象ページ集合
`currentPhase = n` のとき、検索対象は以下。
- Phase0 〜 Phase n の全ページ
- Phase n+1 の `(1)` のみ

例: `currentPhase = 2`
- Phase0 全体
- Phase1 全体
- Phase2 全体
- Phase3(1) のみ

## Phase 解放ルール
### 前提
- `(1)` ページは各 Phase の入り口
- そのページを読むことで当該 Phase を本解放する

### ルール
- 閲覧ページが `PhaseX(1)` なら `currentPhase = max(currentPhase, X)`
- `readPages` にそのページを追加する

## 検索結果の順序
- Phase順
- 同一 Phase 内では `(n)` 順
- 同順位ならタイトル昇順でもよいが、原則は JSON の並び順を保持してよい

## 検索結果 UI
- タイトルのみ表示
- 概要は出さない
- 複数件返却を許可

## 検索UIの配置
### 表示する
- 本体サイトの全ページ
- 資料室の全ページ
- Phase5 ページ

### 表示しない
- ブログ全体
- Phase4(31)

## 右下のフェーズ表示
すべてのページに表示する。
例:
- `Phase0 (1)`
- `Phase2 (14)`
- `Phase4 (31)`
