# AWS デプロイ手順

## ドメイン
`fke.nel-app.jp`

---

## 1. ビルド

```bash
npm run build
```
`dist/` フォルダが生成される。

---

## 2. ACM 証明書（us-east-1 必須）

1. AWS Console → **Certificate Manager** → リージョンを **us-east-1** に変更
2. 「証明書をリクエスト」→ パブリック証明書
3. ドメイン名：`fke.nel-app.jp`
4. 検証方法：DNS 検証
5. Route 53 で「レコードを作成」ボタンをクリック
6. ステータスが **発行済み** になるまで待つ（数分〜10分）

---

## 3. S3 バケット

1. **S3** → 「バケットを作成」
   - バケット名：`fke-nel-app` など（任意）
   - リージョン：`ap-northeast-1`（東京）推奨
   - **パブリックアクセスをすべてブロック**：ON のまま
2. バケット作成後、`dist/` の中身をすべてアップロード
   - `index.html` がルートに来るように
3. 静的ウェブサイトホスティングは**有効にしない**（CloudFront 経由のみ）

---

## 4. CloudFront ディストリビューション

1. **CloudFront** → 「ディストリビューションを作成」

| 項目 | 値 |
|---|---|
| オリジンドメイン | S3 バケットを選択 |
| オリジンアクセス | **Origin Access Control（OAC）** を作成して設定 |
| ビューアープロトコルポリシー | Redirect HTTP to HTTPS |
| 許可された HTTP メソッド | GET, HEAD |
| キャッシュポリシー | CachingDisabled |
| 代替ドメイン名（CNAME） | `fke.nel-app.jp` |
| SSL 証明書 | 上記 ACM 証明書を選択 |
| デフォルトルートオブジェクト | `index.html` |

2. **エラーページの設定**（SPA に必須）
   - 「エラーページ」タブ → 「カスタムエラーレスポンスを作成」
   - 403 エラー → レスポンスページ `/index.html`、HTTP ステータス `200`
   - 404 エラー → レスポンスページ `/index.html`、HTTP ステータス `200`

3. OAC 作成後、「バケットポリシーをコピー」→ S3 バケットのバケットポリシーに貼り付け

---

## 5. Route 53

1. **Route 53** → ホストゾーン → `nel-app.jp` を開く
2. レコードを追加：

| レコード名 | タイプ | 値 |
|---|---|---|
| `fke` | A（エイリアス） | CloudFront ディストリビューション |

エイリアスのルーティング先で「CloudFront ディストリビューション」を選択。

---

## 6. 動作確認

- `https://fke.nel-app.jp/` → トップページが表示される
- 任意のパスを直接開いても正しく表示される（SPAルーティング）

---

## ファイル更新時

```
CloudFront → ディストリビューション → 無効化タブ
→「無効化を作成」→ パス: /*
```
